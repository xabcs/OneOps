# 快速测试指南

## 验证后端不可用处理功能

### 🎯 测试目标
验证当后端服务不可用时，前端应用只显示纯净的登录页面（不包含侧边栏、顶部菜单等主界面元素）。

### 📋 测试步骤

#### 测试1：后端未启动时的行为

1. **确保后端未运行**
   ```bash
   # 停止所有后端进程
   pkill -f "go run main.go"
   pkill -f "main"
   ```

2. **访问前端应用**
   - 打开浏览器访问：`http://localhost:5173/`
   - 或直接访问任何页面，如：`http://localhost:5173/servers`

3. **预期结果** ✅
   - 页面自动跳转到 `/login`
   - **只显示登录页面**（无侧边栏、无顶部菜单、无标签页）
   - 页面顶部显示黄色警告框：
     ```
     ⚠️ 后端服务不可用
     无法连接到服务器，请确认：
     • 后端服务已启动（运行在端口 8082）
     • 网络连接正常
     • 防火墙设置正确
     登录后系统会自动重试连接
     ```

#### 测试2：在不可用状态下尝试登录

1. **在登录页输入凭据**
   - 用户名：`admin`
   - 密码：`123456`
   - 点击"立即登录"按钮

2. **预期结果** ✅
   - 显示提示："后端服务当前不可用，正在尝试重新连接..."
   - 随后显示详细错误信息："无法连接到服务器，请确认后端服务已启动"
   - **仍然保持在登录页面**
   - **不显示任何主界面元素**

#### 测试3：启动后端服务

1. **启动后端服务**
   ```bash
   cd backend && go run main.go &
   ```

2. **等待后端启动完成**（约5-10秒）

3. **刷新登录页面或重新登录**

4. **预期结果** ✅
   - 如果警告框已消失，显示正常登录表单
   - 登录成功后进入**完整的主界面**（包含侧边栏、菜单、标签页等）

#### 测试4：已登录用户遇到后端故障

1. **确保已正常登录**（后端运行中）

2. **在主界面停止后端**
   ```bash
   pkill -f "go run main.go"
   ```

3. **尝试任何操作**（如点击菜单、刷新页面）

4. **预期结果** ✅
   - 立即重定向到登录页面
   - **主界面完全消失**
   - **只显示登录页面 + 后端不可用警告**

#### 测试5：后端服务恢复

1. **在登录页状态下重新启动后端**
   ```bash
   cd backend && go run main.go &
   ```

2. **等待自动恢复或手动刷新页面**

3. **预期结果** ✅
   - 警告框自动消失（可能需要等待最多30秒）
   - 可正常登录
   - 登录后显示完整主界面

### 🎨 界面验证要点

#### ✅ 正确的表现
- 后端不可用时**只显示登录页面**
- **完全没有侧边栏**
- **完全没有顶部菜单栏**
- **完全没有标签页栏**
- 页面居中显示登录表单
- 顶部显示友好的警告信息

#### ❌ 错误的表现（不应出现）
- 侧边栏仍然可见
- 顶部菜单和用户信息仍然显示
- 标签页仍然存在
- 在主界面框架内显示登录表单
- 页面布局混乱

### 🔧 故障排查

#### 如果看到主界面元素
1. 清除浏览器缓存和 localStorage
2. 刷新页面
3. 检查控制台是否有JavaScript错误

#### 如果没有警告提示
1. 检查是否在正确的URL (`/login`)
2. 清除 sessionStorage：`sessionStorage.clear()`
3. 刷新页面

#### 如果无法重定向到登录页
1. 检查路由守卫是否正常工作
2. 查看浏览器控制台错误信息
3. 确认所有文件都已正确修改

### 🚀 自动化测试命令

```bash
# 测试后端不可用
pkill -f "go run" && echo "✅ 后端已停止，访问 http://localhost:5173 验证"

# 测试后端恢复
cd backend && go run main.go & && echo "✅ 后端已启动，刷新页面验证"

# 检查前端状态
curl -s http://localhost:5173 | grep -q "login" && echo "✅ 前端正常运行" || echo "❌ 前端异常"

# 检查后端状态
curl -s http://localhost:8082/api/login -X POST -H "Content-Type: application/json" -d '{"username":"admin","password":"123456"}' | jq -r '.success' && echo "✅ 后端正常" || echo "❌ 后端异常"
```
