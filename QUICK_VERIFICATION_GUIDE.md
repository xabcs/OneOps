# 快速验证指南

## 🎯 验证目标

确认登录页面在后端不可用时**不显示默认警告**，只在登录失败时提示。

## 📋 验证步骤

### 步骤1：停止后端服务

```bash
# 停止所有Go后端进程
pkill -f "go run"
pkill -f "main"

# 确认后端已停止
curl -s http://localhost:8082/api/login || echo "✅ 后端已停止"
```

### 步骤2：访问登录页面

```bash
# 在浏览器中打开
open http://localhost:5173/
# 或者手动访问: http://localhost:5173/
```

### 步骤3：检查登录页面外观

#### ✅ 应该看到的内容：
- 干净的登录页面
- "欢迎回来"标题
- "请输入您的凭据以访问控制台"副标题
- 用户名和密码输入框
- "立即登录"按钮
- "还没有账号？联系管理员注册"提示

#### ❌ 不应该看到的内容：
- 顶部的大段黄色警告框
- "后端服务不可用"的Alert组件
- 任何关于后端状态的提示信息
- 检查清单或操作指引

### 步骤4：尝试登录（预期失败）

1. 在登录页面输入：
   - 用户名：`admin`
   - 密码：`123456`

2. 点击"立即登录"按钮

3. **预期的错误提示**：
   ```
   ❌ 无法连接到服务器，请确认后端服务已启动（端口 8082）
   ```

4. **提示特征**：
   - 右上角显示的红色错误消息
   - 5秒后自动消失
   - 可以手动点击关闭

### 步骤5：启动后端并验证恢复

```bash
# 启动后端服务
cd backend && go run main.go &

# 等待5-10秒让后端启动
sleep 5

# 测试后端是否可用
curl -s http://localhost:8082/api/login -X POST \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"123456"}' | jq -r '.success'
```

### 步骤6：重新登录（预期成功）

1. 刷新登录页面或直接重新登录
2. 输入相同的凭据
3. **预期结果**：
   - 登录成功
   - 显示完整的主界面（侧边栏、菜单等）
   - 右上角显示："欢迎回来, admin"

## 🔧 故障排查

### 问题：登录页面仍然显示警告

**解决方案**：
```bash
# 1. 清除浏览器缓存
# 在浏览器中按 Cmd+Shift+R (Mac) 或 Ctrl+Shift+R (Windows)

# 2. 清除localStorage和sessionStorage
# 在浏览器控制台执行：
localStorage.clear()
sessionStorage.clear()

# 3. 刷新页面
```

### 问题：登录失败没有错误提示

**解决方案**：
```bash
# 1. 检查浏览器控制台是否有JavaScript错误
# 打开开发者工具，查看Console标签

# 2. 检查Element Plus是否正常加载
# 在控制台执行：
typeof ElMessage
# 应该返回 "function"

# 3. 重新加载页面
```

### 问题：后端启动后仍然无法登录

**解决方案**：
```bash
# 1. 确认后端真正在运行
ps aux | grep "go run"

# 2. 测试后端API
curl -s http://localhost:8082/api/login -X POST \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"123456"}'

# 3. 检查后端日志
cd backend && tail -f logs/app.log
```
