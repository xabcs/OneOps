# 系统内置角色说明

## 📋 内置角色列表

系统初始化时会自动创建以下 **5个内置角色**：

| 角色名称 | 角色编码 | 描述 | 权限范围 |
|---------|---------|------|---------|
| 超级管理员 | `admin` | 拥有系统所有权限 | 全部菜单（仪表盘、资产管理、任务、监控、系统管理、审计） |
| 运维工程师 | `ops` | 负责主机和任务管理 | 资产管理、自动化任务、监控中心 |
| 审计员 | `auditor` | 仅拥有查看权限 | 仪表盘、监控中心（基础查看） |
| 普通用户 | `user` | 系统普通用户，拥有基础权限 | 仪表盘、资产管理（查看） |
| 测试角色 | `test` | 用于测试的角色，拥有部分权限 | 仪表盘、资产管理、自动化任务、监控中心（部分） |

## 🔒 内置角色保护

### 前端保护
- ✅ 删除按钮对5个内置角色**置灰禁用**
- ✅ 点击删除按钮时弹出提示：`"XXX" 是系统内置角色，不能删除`
- ✅ 角色列表"状态"列显示"系统内置"标签

### 后端保护
- ✅ 后端删除接口拦截内置角色删除请求
- ✅ 返回错误信息：`系统管理员角色不能删除`（或对应角色名称）

## 🚀 如何初始化

### 方法1：重新运行后端（推荐）
```bash
cd backend
go run main.go
```
后端启动时会自动检测数据库：
- 如果角色表为空 → 创建5个内置角色
- 如果角色表不为空 → 同步更新内置角色的权限（不会删除现有角色）

### 方法2：手动删除角色表后重启
```sql
-- 如果需要重新初始化角色
DELETE FROM roles;
```
然后重启后端，会自动创建5个内置角色。

### 方法3：清理数据库完全重新初始化
```sql
-- ⚠️ 警告：此操作会删除所有数据
DROP TABLE users;
DROP TABLE roles;
DROP TABLE menus;
DROP TABLE login_logs;
DROP TABLE operation_logs;
DROP TABLE system_event_logs;
```
然后重启后端，会重新创建所有表和初始数据。

- 状态列显示"系统内置"

## ⚠️ 注意事项

1. **不可删除**：5个内置角色无法删除（前后端双重保护）
2. **可修改权限**：可以修改内置角色的菜单权限，但不建议
3. **自动同步**：系统启动时会自动同步内置角色的权限，确保与最新菜单结构一致
4. **权限扩展**：需要新的内置角色时，修改后端 `init.go` 中的 `initRoles()` 和 `syncRoleMenus()` 函数
