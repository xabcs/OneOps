# 后端服务可用性检测功能说明

## 功能概述

实现了一个专业的后端服务可用性检测和处理系统，当后端服务不可用时，自动限制用户只能访问登录页面，并提供友好的用户体验。

## 核心功能

### 1. 智能错误检测
- **网络错误检测**：识别连接被拒绝、网络超时等错误
- **认证错误处理**：自动处理401错误，清除过期凭证
- **错误分类**：区分网络错误、服务器错误、客户端错误等

### 2. 路由守卫增强
- **后端状态检查**：路由切换时检查后端可用性
- **自动重定向**：后端不可用时自动重定向到登录页
- **状态保持**：使用sessionStorage保持不可用状态

### 3. 用户友好提示
- **登录页警告**：显示详细的后端不可用警告信息
- **全局通知**：右上角显示后端状态通知
- **重试机制**：提供一键重试连接功能

### 4. 健康检查服务
- **定期检查**：每30秒自动检查后端状态
- **状态同步**：自动同步状态到Vuex store
- **自动恢复**：后端恢复后自动清除不可用标记

## 使用场景

### 场景1：后端服务未启动
1. 用户访问任何需要认证的页面
2. 系统检测到后端不可用
3. 自动重定向到登录页
4. 显示友好的警告信息

### 场景2：后端服务运行中故障
1. 用户正常使用系统
2. 后端突然不可用
3. 右上角显示状态通知
4. 用户可选择重试连接

### 场景3：后端服务恢复
1. 后端服务重新启动
2. 健康检查自动检测到恢复
3. 清除不可用标记
4. 显示恢复成功消息

## 技术亮点

1. **渐进式增强**：不影响现有功能，只增强错误处理
2. **用户体验优先**：友好的错误提示和操作指引
3. **自动化恢复**：无需手动干预，自动检测和恢复
4. **状态持久化**：使用sessionStorage保持状态一致性
5. **组件化设计**：模块化的错误处理和状态管理

## 配置选项

### 健康检查间隔
可以在 `healthCheck.js` 中修改检查间隔：
```javascript
this.checkInterval = 30000 // 30秒
```

### 自定义错误消息
可以在 `errorHandler.js` 中修改错误提示：
```javascript
const ErrorMessages = {
  NETWORK_ERROR: '自定义网络错误消息',
  // 其他错误类型...
}
```

## 注意事项

1. **生产环境配置**：确保生产环境的CORS和安全设置正确
2. **性能优化**：健康检查间隔不宜过短，避免增加服务器负载
3. **错误日志**：建议集成错误监控服务，收集客户端错误
4. **用户反馈**：定期收集用户反馈，优化错误提示文案
